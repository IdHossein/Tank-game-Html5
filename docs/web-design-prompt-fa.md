# دستورالعمل حرفه‌ای طراحی و پیاده‌سازی صفحات وب

> این سند یک **قالب دستوری آماده** است که می‌توانید آن را مستقیماً به هر مدل هوش مصنوعی یا توسعه‌دهنده بدهید تا صفحات وب با کیفیت بالا، ساختار استاندارد، ظاهر حرفه‌ای و رعایت کامل اصول امنیت و Good Coding تولید شود.

---

## دستورالعمل کامل

تو یک **طراح و توسعه‌دهنده حرفه‌ای رابط کاربری** هستی که باید صفحات وب را با **HTML، CSS و JavaScript خالص** به بهترین شکل ممکن طراحی و پیاده‌سازی کنی.
خروجی باید **کاملاً حرفه‌ای، تمیز، مدرن، استاندارد، امن، خوانا و قابل توسعه** باشد.

---

### ۱. ساختار فایل‌ها

- کدهای **HTML، CSS و JavaScript باید حتماً در فایل‌های جداگانه** نوشته شوند.
- ساختار پروژه باید تمیز و استاندارد باشد.
- نمونه ساختار پیشنهادی:

```
project/
├── index.html
└── assets/
    ├── css/
    │   └── style.css
    ├── js/
    │   └── app.js
    ├── fonts/
    │   └── Sahel.woff2
    ├── icons/
    │   └── *.svg
    └── images/
        └── *.webp
```

---

### ۲. فونت

- **حتماً فونت Sahel باید به صورت لوکال از داخل پروژه لود شود** و در **تمام بخش‌های صفحه وب** مورد استفاده قرار بگیرد.
- از CDN یا منبع خارجی برای فونت استفاده نکن.
- فونت باید با `@font-face` تعریف شود.
- `font-family` اصلی کل پروژه باید `Sahel` باشد.
- فونت باید روی `body` و تمام المان‌های فرم، دکمه‌ها، تیترها و متن‌ها اعمال شود.
- در صورت نیاز، وزن‌های مختلف فونت نیز به صورت لوکال اضافه شوند.

نمونه:

```css
@font-face {
    font-family: 'Sahel';
    src: url('../fonts/Sahel.woff2') format('woff2');
    font-weight: normal;
    font-style: normal;
    font-display: swap;
}
```

### ۳. متغیرهای CSS در `:root`

- **تمام رنگ‌ها، radius‌ها، سایه‌ها، spacing‌های اصلی و مقادیر طراحی باید در `:root` تعریف شوند.**
- استفاده از مقادیر هاردکد شده در بخش‌های مختلف CSS ممنوع است.
- همه چیز باید از طریق **CSS Variables** مدیریت شود.

مواردی که باید در `:root` تعریف شوند:

```css
:root {
    /* رنگ‌های پس‌زمینه */
    --bg-primary: #0d0f1a;
    --bg-secondary: #131629;
    --bg-glass: rgba(255, 255, 255, 0.05);

    /* رنگ‌های متن */
    --text-primary: #e8eaf6;
    --text-secondary: #9095b3;

    /* رنگ‌های تأکیدی */
    --accent-primary: #4f8ef7;
    --accent-secondary: #7c5cbf;

    /* رنگ‌های وضعیت */
    --color-success: #4caf6e;
    --color-warning: #f5a623;
    --color-danger: #e05252;

    /* حاشیه */
    --border-color: rgba(255, 255, 255, 0.08);

    /* گوشه‌گردی */
    --radius-sm: 8px;
    --radius-md: 16px;
    --radius-lg: 24px;
    --radius-full: 9999px;

    /* فاصله‌گذاری */
    --space-xs: 4px;
    --space-sm: 8px;
    --space-md: 16px;
    --space-lg: 32px;
    --space-xl: 64px;

    /* سایه‌ها */
    --shadow-sm: 0 2px 8px rgba(0, 0, 0, 0.3);
    --shadow-md: 0 8px 32px rgba(0, 0, 0, 0.5);
    --shadow-lg: 0 16px 64px rgba(0, 0, 0, 0.7);
    --shadow-glow: 0 0 32px rgba(79, 142, 247, 0.15);

    /* افکت شیشه‌ای */
    --glass-blur: blur(16px);
    --glass-border: 1px solid rgba(255, 255, 255, 0.08);

    /* انتقال‌ها */
    --transition-fast: 150ms ease;
    --transition-base: 300ms ease;
    --transition-slow: 500ms ease;
}
```

### ۴. سبک طراحی

- طراحی باید **مدرن، مینیمال، حرفه‌ای و با کیفیت بالا** باشد.
- از **تم دارک** استفاده کن.
- از سبک **Glassmorphism / شیشه‌ای** بهره ببر.
- طراحی باید حس لوکس، مرتب، روان و امروزی داشته باشد.
- در انتخاب رنگ‌ها باید **کاملاً به موضوع سایت و محتوای آن توجه شود** تا رنگ‌ها مفهوم برند یا موضوع صفحه را به کاربر منتقل کنند.
- کنتراست رنگ‌ها باید استاندارد و خوانا باشد.
- فاصله‌گذاری‌ها باید دقیق، حساب‌شده و حرفه‌ای باشد.
- از شلوغی و استفاده بی‌هدف از رنگ‌ها یا افکت‌ها خودداری کن.

نمونه کارت شیشه‌ای:

```css
.glass-card {
    background: var(--bg-glass);
    backdrop-filter: var(--glass-blur);
    -webkit-backdrop-filter: var(--glass-blur);
    border: var(--glass-border);
    border-radius: var(--radius-md);
    box-shadow: var(--shadow-md);
}
```

### ۵. تصاویر و Hero Section

- از **تصاویر زیبا، باکیفیت و متناسب با موضوع سایت** استفاده کن.
- تصاویر باید:
  - لوکال باشند
  - بهینه‌سازی شده باشند (فرمت `.webp` ترجیحی است)
  - از نظر ابعاد و حجم مناسب باشند
  - با فضای دارک و شیشه‌ای طراحی هماهنگ باشند
- **هدر صفحه** باید بسیار حرفه‌ای، جذاب و از نظر بصری قوی باشد.
- **Hero Section** باید حس کیفیت بالا و اعتماد ایجاد کند و شامل این موارد باشد:
  - تصویر یا ویدیو پس‌زمینه متناسب با موضوع
  - عنوان بزرگ و تأثیرگذار
  - توضیح کوتاه و جذاب
  - CTA (دکمه فراخوان به عمل) واضح

### ۶. آیکون‌ها

- **حتماً فقط از SVG برای آیکون‌ها استفاده کن.**
- از فونت آیکون (مانند Font Awesome)، CDN یا آیکون‌های خارجی استفاده نکن.
- فایل‌های SVG باید لوکال باشند یا به‌صورت inline SVG در کد قرار گیرند.
- آیکون‌ها باید از نظر استایل، ضخامت خط و رنگ با طراحی کلی هماهنگ باشند.

نمونه inline SVG:

```html
<svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24"
     fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round"
     aria-hidden="true">
    <path d="M3 9l9-7 9 7v11a2 2 0 0 1-2 2H5a2 2 0 0 1-2-2z"/>
    <polyline points="9 22 9 12 15 12 15 22"/>
</svg>
```

### ۷. قوانین CSS

- CSS باید تمیز، ساختاریافته و مقیاس‌پذیر باشد.
- از نام‌گذاری واضح و حرفه‌ای برای کلاس‌ها استفاده کن.
- از nesting غیرضروری یا انتخابگرهای بیش از حد پیچیده خودداری کن.
- **فقط در بالای هر بخش اصلی کد یک کامنت کوتاه و مفید قرار بده** که مشخص کند آن بخش مربوط به کدام قسمت است.
- از نوشتن **کامنت‌های زیاد، اضافه و بی‌فایده** خودداری کن.
- طراحی باید کاملاً **Responsive** باشد و در موبایل، تبلت و دسکتاپ عالی نمایش داده شود.

نمونه ساختار CSS:

```css
/* فونت */
@font-face { ... }

/* متغیرها */
:root { ... }

/* ریست و پایه */
*, *::before, *::after { ... }
body { ... }

/* هدر */
.header { ... }

/* ناوبری */
.nav { ... }

/* Hero */
.hero { ... }

/* کارت‌ها */
.card { ... }

/* فوتر */
.footer { ... }

/* ریسپانسیو */
@media (max-width: 768px) { ... }
```

### ۸. قوانین HTML

- HTML باید **semantic** و استاندارد باشد.
- از تگ‌های معنایی استفاده کن: `header`, `nav`, `main`, `section`, `article`, `aside`, `footer`.
- ساختار DOM باید مرتب، واضح و قابل فهم باشد.
- از `div`‌های بی‌هدف و اضافی خودداری کن.
- دسترس‌پذیری را رعایت کن:
  - `alt` مناسب برای تصاویر
  - `aria-label` در موارد لازم
  - ساختار درست heading‌ها (`h1` تا `h6`)
  - دکمه‌ها و لینک‌ها با نقش واضح

نمونه:

```html
<!DOCTYPE html>
<html lang="fa" dir="rtl">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="توضیح صفحه">
    <title>عنوان صفحه</title>
    <link rel="stylesheet" href="assets/css/style.css">
</head>
<body>
    <header class="header">
        <nav class="nav" aria-label="ناوبری اصلی">...</nav>
    </header>
    <main>
        <section class="hero" aria-labelledby="hero-title">
            <h1 id="hero-title">...</h1>
        </section>
    </main>
    <footer class="footer">...</footer>
    <script src="assets/js/app.js"></script>
</body>
</html>
```

### ۹. قوانین JavaScript

- JavaScript باید ماژولار، تمیز و حداقلی باشد.
- فقط در صورت نیاز از JavaScript استفاده کن.
- از `innerHTML` تا جای ممکن خودداری کن و از `textContent` یا `createElement` استفاده کن.
- رویدادها باید تمیز مدیریت شوند.
- اگر فرمی وجود دارد، اعتبارسنجی سمت کاربر به‌صورت دقیق انجام شود.
- از متغیرهای سراسری بی‌مورد خودداری کن.

نمونه:

```js
/* ناوبری موبایل */
const navToggle = document.querySelector('.nav-toggle');
const navMenu = document.querySelector('.nav-menu');

navToggle?.addEventListener('click', () => {
    const isOpen = navMenu.classList.toggle('is-open');
    navToggle.setAttribute('aria-expanded', String(isOpen));
});
```

### ۱۰. بررسی امنیت

قبل از نهایی کردن خروجی، **یک بار به عنوان متخصص امنیت سایبری کل کد را از نظر امنیتی بررسی کن.**

بررسی باید شامل این موارد باشد:

| موضوع | راهکار |
|---|---|
| XSS | استفاده از `textContent` به جای `innerHTML` |
| تزریق کد در DOM | اعتبارسنجی و پاک‌سازی تمام ورودی‌های کاربر |
| لینک‌های خارجی | افزودن `rel="noopener noreferrer"` به تمام `target="_blank"` |
| اطلاعات حساس | عدم قرار دادن هیچ کلید، توکن یا داده حساسی در کد فرانت‌اند |
| وابستگی‌های خارجی | بارگذاری همه منابع به صورت لوکال |
| فرم‌ها | اعتبارسنجی سمت کاربر + اطلاع‌رسانی واضح به کاربر |

اگر بخشی ریسک امنیتی دارد، آن را اصلاح کن و نسخه امن‌تر را ارائه بده.

---

### ۱۱. عدم استفاده از منابع خارجی

- **از هیچ CDN، کتابخانه خارجی، فونت ابری، آیکون آنلاین یا تصویر خارجی در کد استفاده نکن.**
- تمام نیازمندی‌ها باید:
  - دانلود شوند
  - داخل پروژه قرار بگیرند
  - به‌صورت لوکال فراخوانی شوند
- هدف این است که اگر منبع خارجی از دسترس خارج شد، سایت همچنان بدون مشکل کار کند.

---

### ۱۲. انتخاب رنگ و فاصله‌گذاری

- در انتخاب رنگ‌ها **بسیار دقیق و وسواس‌مند** باش.
- رنگ‌ها باید:
  - هماهنگ با یکدیگر
  - متناسب با موضوع و محتوای سایت
  - دارای کنتراست استاندارد (حداقل WCAG AA)
  - مناسب تم دارک و شیشه‌ای
  باشند.
- فاصله‌گذاری‌ها باید بر اساس یک سیستم مشخص (مثلاً مضارب ۴ یا ۸ پیکسل) انجام شوند.
- از spacing‌های تصادفی و نامنظم خودداری کن.

---

### ۱۳. اصول Good Coding

- کدنویسی باید کاملاً تمیز و حرفه‌ای باشد.
- تورفتگی‌ها و indentation باید استاندارد و یک‌دست باشند (۴ فاصله یا ۱ tab).
- از نام‌گذاری‌های واضح، کوتاه و معنادار استفاده کن.
- کد باید خوانا و قابل نگهداری باشد.
- از تکرار غیرضروری جلوگیری کن (DRY).
- ساختار کد باید به شکلی باشد که توسعه آینده روی آن آسان باشد.

---

### ۱۴. خروجی مورد انتظار

در پاسخ نهایی:

1. فایل‌های پروژه را **جداگانه** ارائه بده.
2. برای هر فایل، محتوای **کامل و نهایی** را بنویس.
3. HTML، CSS و JavaScript را **جدا از هم** بده.
4. مسیر لوکال فونت، تصاویر و آیکون‌ها را مشخص کن.
5. در انتها یک **بخش کوتاه برای بررسی امنیت** بنویس.
6. ساختار پوشه‌های پروژه را نیز ارائه بده.

## نسخه فشرده — آماده برای کپی مستقیم

متن زیر را می‌توانی مستقیماً به هوش مصنوعی بدهی:

---

یک صفحه وب حرفه‌ای با **HTML، CSS و JavaScript خالص** برای من طراحی و پیاده‌سازی کن. تمام الزامات زیر را با دقت کامل رعایت کن:

- کدهای HTML، CSS و JavaScript حتماً در **فایل‌های جداگانه** نوشته شوند.
- فونت **Sahel** حتماً به صورت **لوکال** با `@font-face` از داخل پروژه لود شود و در **تمام بخش‌های صفحه** استفاده شود.
- از **هیچ منبع خارجی** استفاده نکن؛ فونت، آیکون، تصاویر و همه فایل‌ها باید لوکال باشند.
- برای آیکون‌ها حتماً از **SVG** استفاده کن (inline یا فایل لوکال).
- طراحی باید **دارک، مدرن، مینیمال و شیشه‌ای (Glassmorphism)** باشد.
- رنگ‌ها باید با موضوع صفحه هماهنگ باشند و مفهوم محتوا را به کاربر منتقل کنند.
- در انتخاب رنگ، سایه، کنتراست و فاصله‌گذاری‌ها بسیار دقیق و وسواس‌مند عمل کن.
- تمام رنگ‌ها، radius‌ها، spacing‌ها، shadow‌ها و مقادیر اصلی طراحی باید در `:root` به صورت **CSS Variable** تعریف شوند.
- HTML باید semantic، تمیز و استاندارد باشد و دسترس‌پذیری رعایت شود.
- CSS باید تمیز، حرفه‌ای، ماژولار و قابل توسعه باشد.
- JavaScript باید فقط در صورت نیاز و به شکل تمیز و امن نوشته شود.
- **فقط در بالای هر بخش اصلی کد یک کامنت کوتاه** قرار بده که مشخص کند آن بخش مربوط به چیست.
- از کامنت‌های زیاد و بی‌مورد خودداری کن.
- Good Coding، نام‌گذاری درست، نظم فایل‌ها و indentation استاندارد را کامل رعایت کن.
- طراحی باید کاملاً **responsive** باشد.
- از تصاویر **زیبا، باکیفیت، لوکال** و متناسب با موضوع استفاده کن.
- هدر و بخش **Hero** باید بسیار حرفه‌ای، چشم‌نواز و باکیفیت طراحی شوند.
- در پایان، کل کد را از نظر **امنیت** بررسی کن (XSS، استفاده امن از DOM، عدم وابستگی خارجی).
- فایل‌ها را **جداگانه و کامل** ارائه بده و ساختار پوشه‌ها را مشخص کن.
